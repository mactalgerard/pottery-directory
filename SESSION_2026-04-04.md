### 2. Data Models (`src/models.py`)

Three Pydantic models representing the three pipeline stages:

| Model | Key additions |
|-------|---------------|
| `RawListing` | Direct OutScraper field mapping; `country` injected at ingest time |
| `CleanListing` | Adds `is_verified_niche`, `rejection_reason`, `source_file`, `cleaned_at` |
| `EnrichedListing` | Adds all pottery-specific fields + `description` + `enriched_at` |

All enrichment fields are `Optional` — null over invention is a core rule.

---

### 3. Collect Phase Automation (`src/tools/outscraper_client.py`)

Fully implemented (not a stub). Automates Phase 1 via the OutScraper Google Maps API.

**Search matrix per country:**

| Country | Search terms | Locations | Total queries |
|---------|-------------|-----------|---------------|
| US | 5 terms | 48 metros | 240 queries |
| CA | 5 terms | 30 metros | 150 queries |
| AU | 6 terms (AU-specific) | 20 metros | 120 queries |

AU uses different search terms (`clay classes`, `hand building classes`) because AU studios use those terms more than "pottery wheel".

**Key functions:**
- `collect_listings(country, dry_run)` — main entry point
- `_build_queries(country)` — generates term × location matrix
- `_normalise_row(raw, country)` — maps OutScraper fields → RawListing fields
- `_deduplicate_results(rows)` — deduplicates by `place_id`, then `(name + postal_code)`

Writes to: `data/raw/{COUNTRY}/collect_{date}.csv`

### 4. Pipeline Orchestrator (`pipeline.py`)

Full CLI with four phases:

```bash
python pipeline.py --phase collect              # OutScraper scrape
python pipeline.py --phase collect --dry-run    # Preview queries, skip API
python pipeline.py --phase research             # Validate enrichment fields (LLM)
python pipeline.py --phase clean                # Reject + deduplicate
python pipeline.py --phase enrich               # Enrich via Crawl4AI + Claude
python pipeline.py --to-supabase                # Upsert to Supabase after enrich
python pipeline.py --country AU                 # Scope any phase to a country
```

### 7. Key Design Decisions

- **Country is always explicit** — injected from `--country` flag, never inferred from data
- **Each country is an independent run** — no cross-market mixing of fields or prices
- **Agents never call Crawl4AI directly** — all crawling goes through `src/tools/crawler.py`
- **null over invention** — unconfirmed enrichment fields stay null
- **Fail loud** — Pydantic validation on every model instantiation; EnvironmentError if keys missing
- **Graceful degradation** — crawl failures log and continue; never crash the pipeline
